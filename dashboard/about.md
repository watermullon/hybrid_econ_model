# Hybrid Fund Economic Model

This is a deterministic annual scenario model for a hybrid fund structure. LP capital is deployed across three sleeves — real estate, hedge fund, and reserve — and the model runs year-by-year to test whether the LP achieves their 2x cash return hurdle.

The model is not a Monte Carlo simulator. Each scenario is a single deterministic run with explicit assumptions defined in YAML files. Results are fully reproducible.

---

## Fund structure

The fund raises LP capital and splits it across three sleeves at inception:

- **Real estate** — equity deployed into property deals. The model tracks NOI, debt service, capex, property value, DSCR, and refinance events at the individual deal level.
- **Hedge fund** — liquid sleeve that appreciates annually by a configured return. Gains can be harvested and routed to LP distributions or reinvested.
- **Reserve** — cash buffer that appreciates at a fixed rate. Used to fund acquisitions or support LP distributions if other sleeves fall short.

Cashflow generated by each sleeve is split by a configurable routing policy — for example 60% to LP distributions, 25% reinvested into the hedge fund, 15% to reserve.

---

## How the LP hurdle works

The LP target is a **2x cash MOIC** on their committed capital. Every dollar distributed to the LP reduces the remaining hurdle. The model tracks this year by year.

At eligible years a **hurdle completion trigger** tests whether the fund can close the gap by liquidating the hedge fund, drawing on the reserve, and executing a final refinance. If total available liquidity covers the remaining hurdle, the scenario ends early and the LP is redeemed.

Anything left after LP redemption flows to the GP as residual NAV.

---

## Real estate modelling modes

**Top-down** — a single set of RE assumptions (NOI yield, appreciation rate) applied to the whole sleeve. Used for the generic scenarios.

**Bottom-up** — individual deals modelled explicitly with their own purchase price, debt, NOI ramp, capex schedule, and refinance events. The Jon scenarios use this mode with `jon_deal_1`: $10M purchase, $7M debt at 70% LTV, $3M equity.

## Scenarios

| Scenario | What it tests |
|---|---|
| Moderate RE and HF baseline | Ordinary cash routing reaching the LP hurdle |
| Crypto sleeve strong | Whether liquid HF alpha can accelerate LP returns |
| Slow growth, weak early HF | Long-duration distribution drift |
| HF drawdown, RE survives | RE cashflow covering LP when HF is impaired |
| RE distress, HF strength | Whether HF gains can offset property stress |
| Strong RE and strong HF | Upper-end residual asset outcome |
| High NAV growth, low RE liquidity | Gap between paper value and LP cash |
| Weak RE and weak HF | LP never reaches 2x |
| Jon base case | $10M deal, 13% HF, standard routing |
| Jon downside | Slower lease-up, tighter refi, HF -15% Y1 |
| Jon upside | Faster lease-up, 25% HF |
| Jon property shock 25% | 25% value drop Year 2, 5-year recovery |
| Jon property shock 40% | 40% drop — property underwater, covenant triggered |
| $5M fund | $5M raise, $3M to RE / $2M to HF |

### 2026-06-03
- **Deal resized to $10M at 70% LTV** — corrected from an earlier $13M/$10M debt structure that implied 77% LTV, inconsistent with the 70% refi policy. Now: $10M purchase, $7M debt, $3M equity. NOI and capex scaled proportionally across all jon_* scenario overrides.
- **`five_million_fund` allocation bug fixed** — the allocation function was ignoring `lp_capital_override` and always sizing sleeves off the global $10M LP capital. A $5M fund was incorrectly deploying $6M to RE. Fixed so any scenario with `lp_capital_override` allocates correctly.

### 2026-06-02
- **LTV covenant breach + HF equity injection** — when a deal's property value falls below the configured LTV covenant threshold, the hedge fund automatically injects capital to pay down debt. Tracked as `covenant_hf_injection` in cashflows. The HF appreciates on its reduced post-injection base (bug in original implementation caught and fixed — the injection was previously "free" because appreciation used the pre-injection NAV).
- **Property shock scenarios** — added `jon_property_shock_25` (−25% value drop Year 2, 5-year recovery) and `jon_property_shock_40` (−40% drop, property deeply underwater). Uses per-year value growth overrides (`value_growth_by_year`) on deal valuation.
- **`five_million_fund` scenario** — $5M raise with fixed allocation: $3M to RE, $2M to HF, no reserve. Uses new `lp_capital_override` field on `Scenario` to size the LP hurdle and sleeve allocations independently of the global model config.
- **Dashboard: covenant breach event panel** — the Simulator annotates years where the HF injected equity, showing breach LTV, old/new debt balance, injection amount, and debt service saving.
- **Dashboard: NAV bridge fix for covenant injection** — HF appreciation line in the NAV bridge correctly excludes the covenant injection from the compounding base.
